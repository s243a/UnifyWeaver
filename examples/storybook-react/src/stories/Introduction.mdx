# UnifyWeaver Visualization Components

Welcome to the Storybook for UnifyWeaver generated visualization components. This documentation showcases the React components generated by the UnifyWeaver Prolog-to-code generation system.

## Generating Components

Components can be regenerated from Prolog specifications:

```bash
cd examples/storybook-react
swipl generate.pl
```

This generates components in `src/components/generated/` from the UI primitives defined in `src/unifyweaver/ui/`.

## Overview

UnifyWeaver generates production-ready visualization components from declarative Prolog specifications. The generated components include:

### Themes & Styling
- **Theme System** - Centralized theme definitions with CSS custom properties
- **Animation Presets** - Entry, exit, and attention animations
- **Template Library** - Pre-built dashboard and report templates

### Responsive Layouts
- **Breakpoint System** - Mobile, tablet, desktop responsive grids
- **Container Queries** - Component-level responsiveness
- **Adaptive Cards** - Content that adapts to container size

### Interactions
- **Tooltips** - Hover-triggered contextual information
- **Pan & Zoom** - Navigate large visualizations
- **Drill-Down** - Explore hierarchical data
- **Selection** - Single and multi-select with keyboard support

### Performance
- **Virtual Scrolling** - O(1) render time for 10,000+ items
- **Lazy Loading** - Intersection Observer-based pagination
- **Chunked Loading** - Progressive data processing
- **Memoization** - React.memo optimization patterns

### Charts & Visualizations
- Line, Bar, Pie, Area charts
- Radar, Funnel, Gauge, Sankey, Chord diagrams
- Heatmaps and Treemaps
- 3D plots and surfaces

## Getting Started

1. Generate components from Prolog specifications:

```prolog
% Generate a themed line chart
?- generate_visualization(my_chart, line_chart, [
    theme(dark),
    responsive(true),
    animation(fade_in)
], Code).
```

2. Copy generated code to your React project
3. Import and use components
