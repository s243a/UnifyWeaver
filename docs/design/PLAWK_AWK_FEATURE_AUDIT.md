# plawk ‚Äî AWK feature audit & roadmap

**Status**: living checklist (audited 2026-07). plawk is a *specialised* awk-like
front-end for the hybrid WAM/LLVM target, not a full AWK clone ‚Äî it adds things
POSIX AWK has no notion of (multi-pass caches, `over query(Goal)` readers,
generator blocks, durable LMDB stores, a Prolog foreign-call surface). This doc
tracks how much of the *basic* AWK surface exists, so the DSL stays familiar to
AWK users, and prioritises the gaps. Legend: ‚úÖ done ¬∑ ‚óê partial ¬∑ ‚è≥ surface
only (runtime pending) ¬∑ ‚ùå missing.

## Patterns

| Feature | Status | Notes |
|---|---|---|
| `BEGIN` / `END` | ‚úÖ | incl. constant `print` (BEGIN/END literal print) |
| `/regex/` | ‚úÖ | a bare `/re/` pattern matches the whole record with **full POSIX ERE**: a body with metacharacters lowers to `field_match(0, Regex)` and runs through the same regex engine as `~`/`!~` (`/foo.*bar/`, `/^(err\|warn):/`, character classes, alternation, groups). A metachar-free body keeps its fast native lowering (`^prefix` ‚Üí `prefix/1`, all-literal ‚Üí `contains/1`); `\/` is a literal slash |
| `$N == "v"`, `$3 > 100` | ‚úÖ | field-equality + numeric field guards |
| `&& \|\| !` combinators | ‚úÖ | awk precedence, parens, single-block lowering |
| `~` / `!~` match | ‚úÖ | POSIX ERE |
| expression pattern (bare `NR==1`) | ‚óê | comparison guards yes; arbitrary expr no. **`NF OP int` and `length OP int` are bare rule patterns** (`NF > 3 { ‚Ä¶ }`, `NF == 0 { ‚Ä¶ }`, `length > 80 { ‚Ä¶ }`, `length($0) <= 40 { ‚Ä¶ }`) ‚Äî they parse to `special_cmp(Special, Op, Value)` (`Special` = `'NF'` or `length`) and lower through the pattern-guard path (NF by counting fields via `@wam_atom_field_count_value` with the active FS; `length` by `strlen` of `$0`), then `icmp` against the literal, so they compose with `!`/`&&`/`||` and the other base patterns (`NF >= 2 && $1 == "x"`). **Both operand orders parse** ‚Äî `SPECIAL OP int` and the reversed `int OP SPECIAL` (`3 < NF`, `5 >= length`), which swaps the operator. **Field-vs-field comparison is a pattern too** ‚Äî `$I OP $J { ‚Ä¶ }` (`$1 > $2`, `$2 == $3`) parses to `field_cmp2(I, Op, J)` and compares the two field slices by POSIX strnum rules (numeric when both look numeric, else lexical) via `@wam_strnum_cmp_slices` ‚Äî a slice-aware comparator that NUL-terminates the two field spans in a reusable, geometrically-grown scratch buffer, so it does **not** intern per-record values (constant-memory streaming preserved); a field beyond `NF` compares as the empty string, and an invalid-slice failure is masked so it cannot satisfy `!=`/`>`/`>=`. **Arithmetic-expression patterns** ‚Äî `$I ARITH K CMP int { ‚Ä¶ }` (`$1 + 0 > 5`, `$1 % 2 == 0`, `$1 * 2 >= 10`, `$1 - 1 == 0`) parse to `field_arith_cmp(I, ArithOp, K, Op, RHS)` and lower by parsing field `I` as a signed i64 (non-numeric ‚Üí 0, matching plawk field arithmetic), applying the integer op (`+`/`-`/`*` and `%` = `srem`, nonzero divisor enforced at parse), then `icmp` against the literal; composes with the combinators. **Field-vs-field arithmetic is a pattern too** ‚Äî `$I ARITH $J CMP int { ‚Ä¶ }` (`$1 + $2 > 100`, `$2 - $1 == 0`, `$1 * $2 >= 50`) parses to `field_field_arith_cmp(I, ArithOp, J, Op, RHS)` and lowers by parsing both fields as signed i64 (each non-numeric ‚Üí 0), applying the integer op to the two field values, then `icmp` against the literal; only `+`/`-`/`*` are accepted (a `%`/`/` with a *field* divisor, whose zero-divisor is UB / an awk fatal, is cleanly rejected at parse). **Float division is a pattern too** ‚Äî `$I / K CMP V { ‚Ä¶ }` (`$1 / 2 > 3.5`, `$1 / 3 <= 10`, `$1 / 100 >= 0.5`) parses to `field_div_cmp(I, K, Op, RHS)` and lowers as a *floating-point* op (unlike the integer arithmetic ops): the field is parsed as an f64 (`@wam_atom_field_f64_value`, non-numeric ‚Üí 0.0), divided by the integer divisor `K` (a `K.0` double literal), then `fcmp`'d against the RHS ‚Äî carried as `float_const(M, D)` whether written as a float (`3.5`) or an integer (`10`), built as the exact rational `fdiv M.0, D.0` (the inexact-decimal-safe idiom). The divisor must be a nonzero integer literal (parser-enforced); a field or float divisor is a follow-on (a zero field divisor would give `inf`, diverging from awk's fatal). **A bare scalar-variable pattern landed** ‚Äî `NAME OP int { ‚Ä¶ }` (`n > 2`, reversed `2 < n`) parses to `scalar_cmp(Name, Op, Value)`; the codegen resolves `Name` to the rule's current slot SSA value (rule 0: the loop-head phi `%slot_M`; rule N: the per-rule input phi `%rule_N_in_slot_M`, which dominates the guard) via a pattern-AST rewrite at the rule-chain call site (`plawk_resolve_scalar_cmp/4`), then emits an i64 `icmp` ‚Äî so the pattern-guard predicate stays un-threaded and it composes with `!`/`&&`/`||` for free. The scalar must be an i64 (counter) slot, i.e. assigned somewhere so it has a slot; a never-assigned scalar (no slot) or a double/string scalar is cleanly declined at compile time (conservative v1 ‚Äî awk treats an unset scalar as 0). **A field compared to a scalar var landed on this plumbing** ‚Äî `$I OP NAME { ‚Ä¶ }` (`$1 > threshold`, reversed `threshold < $1`) parses to `field_scalar_cmp(Index, Op, Name)`, resolves `Name` to the rule's slot value the same way, and reuses the `$I OP int` numeric field-comparison runtime (`@wam_atom_field_i64_cmp_value`, which takes the expected value as a runtime i64 argument ‚Äî the scalar's SSA value passes straight in), so the semantics match the field-vs-int-literal pattern exactly (a non-numeric/missing field makes the comparison false). Both operand orders; composes with the combinators. **A bare string-scalar pattern landed too** ‚Äî `NAME OP "literal" { ‚Ä¶ }` (`name == "alice"`, `tag != "skip"`, ordering ops, reversed `"alice" == name`) parses to `scalar_str_cmp(Name, Op, Str)` and resolves `Name` against a string-valued slot ‚Äî either `scalar_string` (literal-assigned) or `scalar_strnum` (field-assigned, e.g. `s = $1`) ‚Äî comparing by interned atom id (`==`/`!=`, canonical so equal strings share an id) or `strcmp` (ordering, id‚Üítext with an unset id 0 ‚Üí empty), reusing the scalar-`if` string-guard lowering; composes with the combinators. Comparing a strnum against a string *literal* is a string comparison in awk, which the canonical-id/strcmp path matches exactly. **A field compared to a string/strnum scalar landed too** ‚Äî `$I OP NAME { ‚Ä¶ }` where `NAME` is a string (literal-assigned) or strnum (field-assigned, `key = $1`) scalar: the field-vs-scalar resolver, when `NAME` is not an i64 counter, compares the field slice against the scalar's stored string (`@wam_atom_to_string` of its id, unset id 0 ‚Üí empty) via the same POSIX strnum SLICE comparator as `$I OP $J` (numeric when both look numeric ‚Äî `$2 == key` matches `10` vs `10.0` ‚Äî else lexical); both operand orders, composes with the combinators. The one rare divergence: a string-*literal*-assigned scalar with a numeric-looking field compares strnum-numerically rather than as a true string. Follow-on: a double scalar operand (`$1 > rate`). **`NR` is a bare rule pattern too** ‚Äî `NR OP int { ‚Ä¶ }` (`NR == 1`, `NR > 2`, and the reversed `2 < NR`; `FNR` is the alias) parses to `special_cmp('NR', Op, Value)` and `icmp`s the record counter `%current_nr` against the literal, composing with `!`/`&&`/`||` (windows like `NR >= 2 && NR <= 3`); the counter is emitted only when a pattern (or scalar guard / print) actually uses NR, so non-NR programs keep the streaming counter elided. **`NR` is a special in `if`/`while` condition guards** (`if (NR == 2)`, `if (NR >= 2 && NR <= 3)`) ‚Äî the record counter, not a phantom scalar slot (a prior bug read it as an uninitialised var ‚Üí always 0). Fixes the record-gated idioms (`if (NR == 1) { first = $1 }`, conditional accumulation). **`NF` is a guard special too** (`if (NF > 3)`, `if (NF >= 2 && NR == 1)`) ‚Äî the field count of the current record, computed from `%line` via the field-count runtime with the active `FS` threaded into the condition lowering; works in `if`/`while`/`do-while` guards. `NF` in an END condition is a clean not-yet (no current record) |
| range pattern (`/a/,/b/`) | ‚úÖ | `start, end { ‚Ä¶ }` fires for records from a `start` match through an `end` match (inclusive), via a per-rule i1 latch global; re-arms for later ranges, runs to EOF if unterminated. **Endpoints are full patterns**, each reusing the ordinary pattern-guard lowering (which recurses through the combinators): regexes (`/S/,/E/`), expression patterns (`NR==1, NR==3`), field comparisons, `!`/`&&`/`||` combinators (`NR>=2 && /x/, /end/`, `/a/ || /b/, /z/`), and any mix (`NR==2, /end/` or `/b/, NR==4`). A single pattern with no comma is unaffected (falls through to the ordinary rule) ‚Äî an `&&`/`||` chain stops at the top-level comma, so the endpoint separator is never consumed |

Counter tables (`arr[k]++`, a genuine i64) keep the plain icmp for integer RHS. **Non-field strnum sources landed**: a scalar from a command-line argument (`x = ARGV[N]`) or a getline read (`getline v < "file"`, `s = getline v < "file"`) is a strnum too ‚Äî `if (x > 5)` dispatches through `@wam_strnum_cmp_int` on its text (previously it compared the interned atom id ‚Äî every value looked "big", a bug). A candidate whose reads are unsupported (e.g. used in a concat) deactivates to a plain string scalar (its prior behaviour), so activation never regresses. Remaining: general rule-body `arr[N]` element reads (outside the compilable surface ‚Äî only the for-in contexts read elements); nested-block field copies (blocked on a pre-existing nested-assignment control-flow limitation). | `getline` | ‚óê | **Main-input, redirected, and command-pipe scalar/record getline landed.** Plain `getline` reads the next record from the primary driver stream into `$0`, re-splits `$1‚Ä¶$NF` with the active FS, updates `NF`, and advances `NR`/`FNR`; `getline var` advances the same stream and counters but preserves `$0`/fields. Both honor the active RS (including regex RS) and RT because they share the driver's reader state. Redirected `getline var < "file"` and `getline < "file"` use filename-keyed handles; redirected record getline does **not** advance NR/FNR and remains physical-newline-only in v1 while preserving RS/RT. Pipe reads are physical-newline-only and preserve RT in v1; updating RT from the matched pipe separator remains a follow-on. All three families support bare statements, status capture, and canonical while-drain forms, returning 1 on a record, 0 at clean EOF (including a child that merely exits nonzero), or -1 on an actual pipe/open/read/close/allocation failure; EOF/error preserve the target. `cmd | getline` necessarily executes the shell command supplied by the awk program. ## Functions

| Feature | Status | Notes |
|---|---|---|
| user functions (`function f(a) { return ‚Ä¶ }`) | ‚úÖ | compile to Prolog clauses; work in accumulator / typed-field (`BINFMT`) contexts **and text mode** ‚Äî `print f($1)` auto-coerces the field (awk semantics). Optional numeric arg typing (`function f(num x)`) skips the coercion (typed-fast path). |
| string: `length` `substr` `index` `tolower` `toupper` | ‚úÖ | native, allocation-free. **Bare `length`** (no argument) is awk shorthand for `length($0)` ‚Äî it parses to the same `length(field(0))` AST, so it works in print, arithmetic (`length + 1`), and scalar assignment (`n = length`); `length(...)` / `length (...)` stay calls and `lengthy` stays an identifier. **`substr` takes 2 or 3 args**: `substr(s, m, n)` (the m..m+n-1 slice) and `substr(s, m)` (the tail from m to end) ‚Äî the 2-arg form lowers to a `to_end` marker that the runtime slice clamps to the bytes remaining after m |
| string: `split` `sub` `gsub` `match` `sprintf` | ‚óê | **`sprintf` + `split` landed.** `sprintf`: `x = sprintf("fmt", args)` ‚Üí string scalar (printf engine + `snprintf`‚Üíintern). `split`: `split($N, arr, "sep")` populates a positional string array `arr` (keys 1..n) via `@wam_str_split_into` (clears + repopulates; empty pieces kept), read via for-in (`for (k in arr) print k, arr[k]`). the separator may be a **single char (literal byte) or a multi-char POSIX ERE regex** ‚Äî a per-split-site pattern/cache (`@wam_str_split_into_re`), independent of `FS`. Follow-ons: a return count, a default-FS separator, and `arr[N]` constant-index read. **`sub`/`gsub`/`match` landed.** `match(SRC, /re/)` (SRC = `$0` or `$N`) returns the 1-based match position (0 if none) and sets **`RSTART`/`RLENGTH`** (`RLENGTH` = -1 on a miss) via `@wam_regex_match` ‚Äî an i64 expression usable in `print`. `sub`/`gsub`: `Pattern { sub/gsub(/re/, "repl"); ‚Ä¶ ; print $0 }` rewrites `$0` (first match / all matches) via `@wam_regex_gsub`, with an unescaped `&` in the replacement expanding to the matched text and a per-site compiled ERE; a string-literal pattern works too, and chained substitutions compose. **Scope:** `match` works in `print` **and scalar-assignment** contexts (`n = match($0, /re/)`, `x = RSTART`); and `RSTART`/`RLENGTH` read as i64 specials in `if`/`while` comparison guards (`if (RSTART > 0)`, `if (RLENGTH >= 3)` ‚Äî the special is loaded, not a phantom slot). `sub`/`gsub` target `$0` (the stream-editor idiom), a **string-scalar variable** (`gsub(/re/, "repl", s)` ‚Äî substitutes over the slot's current interned value and re-interns), **or a field `$N`** (`gsub(/re/, "repl", $2)` ‚Äî rewrites the field via the field buffer and rebuilds `$0` with OFS; composes with `$N = expr` assignments in the same record). **Count capture** landed too: `n = gsub(/re/, "repl", var)` (and `sub`) is a dual-slot write ‚Äî `var` is substituted in place (a string scalar) and `n` receives the substitution count (an i64 slot, loaded from the shared `@plawk_gsub_count`), usable in later guards (`if (n > 1)`). All compose with chaining, `&`, `END`, and string-equality guards. Follow-ons: `match` as a bare guard condition (`if (match(...))` ‚Äî use `if ($0 ~ /re/)` or capture-then-test), and count capture into a field target (`n = gsub(..., $2)`) |
| numeric: `int` `sqrt` `sin` `cos` `exp` `log` `atan2` `rand` `srand` | ‚úÖ | `int` (truncation) plus the **libm math builtins**: `sqrt`/`sin`/`cos`/`exp`/`log` (unary), `atan2` (binary), and **`rand`** (zero-arg). Each parses to `math_call(Fn, Args)` and lowers through the double expression path ‚Äî the argument is evaluated in f64 (an integer field promotes via `sitofp`, so `sqrt($1)` works), the libm function is called, and the result prints via `%g` (`sqrt(16)` ‚Üí `4`, `sqrt(2)` ‚Üí `1.41421`). `log` is the natural logarithm. `rand()` returns a double in `[0, 1)` (libm `drand48`). Arguments may be full arithmetic expressions (`sqrt($1 * 4)`, `sqrt(2^4)`), and a call composes inside larger arithmetic (`2 * sqrt(9)`, `rand() * 100`) and `printf`. **`srand(N)` / `srand()`** are rule-body seeding statements (`@srand48`): `srand(N)` makes the sequence reproducible (drand48 is POSIX-specified, so `srand(1)` ‚Üí `rand()` = `0.0416303`), `srand()` seeds from the wall clock. A name is only a builtin when directly followed by `(`, so `sine`/`randy` stay identifiers. Same supported surface as exponentiation (print / printf / arithmetic value positions); guards and double-scalar-reread are the same pre-existing limitations. Follow-ons: `srand` in a `BEGIN` block (BEGIN hosts no non-print action yet), a variable/expression seed (`srand($1)`), and `srand`'s awk return value (the previous seed) |
| Prolog foreign calls (`pred($1)`, `float(pred(‚Ä¶))`) | ‚úÖ | plawk-specific, beyond AWK |

## Variables & operators

| Feature | Status | Notes |
|---|---|---|
| `$0` `$N` `NR` `NF` `FS` `OFS` | ‚úÖ | `FS` accepts a **multi-char / regex** value (a POSIX ERE, awk semantics): a length-‚â•2 `BEGIN { FS = "‚Ä¶" }` compiles to a reserved sentinel separator byte that the field runtime dispatches to `@wam_fs_regex_field_slice_value` / `_count_value` (lazily `regcomp`ed, one FS per program). Because the numeric / `length` / `eq` / `cmp` field projectors all delegate to the core slice, the regex FS reaches `$N` reads, `NF`, guards, and concat uniformly; `$0` and EOF are unaffected. A single-char `FS` stays a literal byte (awk treats a one-char FS literally). **Field assignment splits on a regex FS too** (via `@wam_fields_new_re`), and `split()` takes its own multi-char/regex separator. **`OFS` is now multi-char / empty**: it is a list of bytes (default `[32]`), emitted between a print's fields as one `putchar` per byte, so `BEGIN{OFS=", "}` prints `a, b` and `OFS=""` prints fields adjacent; it composes with `ORS`. | `ARGV[N]` `ARGC` | ‚óê | **command-line arguments landed** (rule bodies): the plawk driver is a native binary, so `ARGV`/`ARGC` read straight from the process `argv` via `/proc/self/cmdline` (`@wam_argc`, `@wam_argv_get`) ‚Äî `ARGV[0]` the program, `ARGV[1]` the first argument (the input path; `-` = stdin), `ARGV[2..]` extras, matching awk. `ARGC` is an i64 usable anywhere an i64 special is (`print ARGC`, `if (ARGC > 1)`, `n = ARGC - 1`, `c = ARGC`, concat). `ARGV[N]` (a non-negative integer-literal index) is a string scalar: `print ARGV[N]`, `x = ARGV[N]` (then prints / string-compares); an out-of-range index is `""`. Linux-specific (`/proc/self/cmdline`) |
| `FNR` `FILENAME` | ‚óê | **landed as single-input record-identity specials.** plawk streams one input, so there is no per-file counter reset: **`FNR` is an exact alias for `NR`** (the record number) ‚Äî it reads anywhere `NR` does (print, arithmetic, `if`/`while` guards, `END`), because it parses to `special('NR')` and reuses the NR codegen unchanged. **`FILENAME` is the input path** ‚Äî the native driver takes the input as `ARGV[1]` (`-` denotes stdin), so `FILENAME` parses to `argv_at(1)` and reuses the `ARGV[N]` string machinery: a string scalar for `print FILENAME`, `x = FILENAME`, and string-equality via assign-then-compare (`f = FILENAME; if (f == "path")`). Follow-ons: `FILENAME` in an `END` block (same limitation as `ARGV[N]` in `END`); a direct `FILENAME`/`ARGV[N]` comparison operand |
| `ORS` | ‚úÖ | **output record separator, any string.** `BEGIN { ORS = "‚Ä¶" }` sets what is printed after each `print` (default newline) ‚Äî single-char, multi-char (`" | "`), or empty. It is emitted through a runtime pointer global `@plawk_ors_ptr` (statically initialised to the ORS constant) that every print terminator loads and prints via `printf("%s", ptr)`, so the ORS may be any length and any content ‚Äî **a literal `%` is data, not a format directive** ‚Äî with no change to the terminator emitters and no per-emitter length. Per-record **and** `END` prints honour it, and it composes with (multi-char) `OFS` (`OFS` separates a print's fields, `ORS` terminates the record ‚Äî `BEGIN{OFS="-";ORS=" | "} {print $1,$2}` ‚Üí `a-b | c-d | `) |
| `RS` | ‚úÖ | **input record separator: literal char, POSIX-ERE regex (gawk), or paragraph mode.** `BEGIN { RS = "‚Ä¶" }` sets what the readers split on. A one-character `RS` is a literal byte (`";"`); a **nonempty multi-character `RS` is a POSIX extended regular expression** (gawk's regex-RS extension), so variable-width separators, alternation, groups, and character classes work ‚Äî `BEGIN{RS="[[:digit:]]+"}` over `foo12bar345baz` yields `foo`/`bar`/`baz`. The runtime dispatches on `@wam_rs_mode` (0 literal / 1 paragraph / 2 regex): a regex `RS` is `regcomp`ed once at startup, and each reader scans the buffered record with `regexec`, trims the leftmost **nonempty** match, and replays the bytes after it. The exact matched separator is exposed through read-only **`RT`** (including in `END`; cleared on an unterminated record). Both readers (`@wam_stream_read_line_transient_value` and its persistent `getline` sibling) share the behaviour, and it composes with a regex `FS`. The trailing-CR strip (CRLF) is gated on `RS` being exactly a one-byte newline, so a custom `RS` keeps `\r` as data. **An empty `RS` is paragraph mode:** each reader skips leading blank lines, treats two adjacent newlines as the boundary, collapses blank-line runs, and strips a lone trailing newline at EOF ‚Äî a paragraph may span lines, and the default whitespace FS splits `$N` across the embedded newlines. An invalid ERE fails at runtime (exit 2, focused diagnostic). Caveat: POSIX `regexec` has no partial-match interface, so a match that begins but stays incomplete at a read-buffer boundary follows the documented once-per-block scan rather than once-per-byte |
| `SUBSEP` | ‚óê | **multi-dimensional subscripts landed** for the two-field key. `arr[$i,$j]` joins the two field values with SUBSEP into one string key (awk `arr[i SUBSEP j]`): the write counter `arr[$i,$j]++`, the element read `arr[$i,$j]`, **membership `($i,$j) in arr`**, and for-in over the joined keys. All three key-building paths call `@wam_intern_subsep_key2` (slice both fields ‚Üí join with the SUBSEP bytes ‚Üí intern to one atom id), keyed off `@wam_subsep_ptr`/`@wam_subsep_len` (default the `0x1C` byte, awk's SUBSEP). `BEGIN { SUBSEP = "‚Ä¶" }` overrides it (any string, incl. empty = no separator) via the RS-style begin-globals + begin-instructions seam. Follow-ons ‚Äî **clean rejection, never miscompiled**: three-plus subscripts / membership tuples (`arr[$i,$j,$k]`, `($i,$j,$k) in arr`), non-field multi-dimensional components (`arr[$i,"x"]`), and multi-dim on the other write shapes (`=`, `+=`, `delete`) / binfmt records |
| `ENVIRON["NAME"]` | ‚óê | **read-only env lookup landed** (getenv via `@wam_environ_get`): `x = ENVIRON["NAME"]` (a string scalar ‚Äî so it prints and string-compares) and `print ENVIRON["NAME"]`; an unset variable is `""` (awk semantics). v1 takes a string-literal key. Follow-ons: a non-literal key, `for (k in ENVIRON)`, and ENVIRON in juxtaposition-concat |
| `RSTART` `RLENGTH` | ‚úÖ | set by `match(SRC, /re/)`, readable as i64 specials in `print` (backed by `@plawk_rstart`/`@plawk_rlength`) |
| arithmetic `+ - * / % //` | ‚úÖ | i64, awk precedence, safe div/mod |
| comparison, `~`/`!~` | ‚úÖ | |

Scalar-var operands, string branches, and `&&`/`||` conditions are follow-ons |
| string concatenation (juxtaposition `$1 $2`) | ‚úÖ | `print` context **and** assignment: `print $1 $2`; `x = $1 $2` / `x = "id:" $1` build a **string-valued scalar** (an interned atom id in an i64 slot, resolved to text on read/print), **incl. accumulation `x = x $1`** (a string-scalar read as a concat operand ‚Äî resolved to text and re-interned). Arithmetic binds tighter, comma still splits |
| exponentiation `^` / `**` | ‚óê | **landed for expression / output contexts.** `^` and `**` are the same operator ‚Äî a new precedence level that binds tighter than `* / %` and is **right-associative** (`2^3^2` == `2^(3^2)` == 512). Operands may be integers, fields, or floats (`$1^2`, `2.5^2`), and it composes inside larger arithmetic (`$1^2 + 1`, `10 - 2^3`), juxtaposition concat (`print "r=" 2^3`), and `printf` `%g`/`%e`/`%f`. ## Prioritised gaps (recommended order)

1. **`while` / `do-while` loop runtime ‚Äî LANDED (PR 2).** The loop iterates its
   mutable scalar state via **loop-header phis** (reusing `foreach_loop`'s head
   phis ‚Äî no memory slots needed after all; see `PLAWK_CONTROL_FLOW_PLAN.md`).
   Body: `set` / `inc` / `+=` over i64 + `print`. Enablers landed with it:
   **bare scalar-var print** (`print i`) and a **body-printing scalar chain with
   no `END`**. **PR 3 (general condition) LANDED**: scalar comparisons
   `VAR CMP (int | VAR)` combined with `&&` / `||`. **PR 3b (`while`
   break/continue) LANDED**: SSA merge phis at the loop exit (`break`) and head
   (`continue`); scalar `if` conditions and END scalar-`if` landed alongside.
   **do-while break/continue and nested loops also LANDED** (`while` inside
   `while`/`do-while`, inner break targets the innermost loop). Remaining: loops
   inside a multi-pass `pass { }` block (PR 4).
2. **User-function call in text/print context ‚Äî LANDED (auto-coerce).**
   `print f($1)` used to return `0`: a text field reached the synthesised
   `f(X,R) :- R is X*2` as an *atom*, failing `is`. Now the synthesised clause
   coerces each untyped param inline ‚Äî `(number(H) -> V = H ; atom_number(H, V))`
   ‚Äî so a text field parses to a number (awk semantics) while a typed i64 arg
   (`BINFMT`) passes straight through. No engine change, no `num($1)` required.
   *Typed-fast path ‚Äî LANDED (optional arg typing).* An optional numeric type
   annotation, `function f(num x)` (also `int` / `float`), declares the param
   already-numeric, so the synthesised clause **skips the coercion goal** ‚Äî the
   head var *is* the value var. Because plawk compiles to **typed WAM**, a
   declared arg carries in its native representation and the hot path pays
   nothing (auto-coerce otherwise runs `number/1` every call). This is the same
   principle as explicit `emit` (`PLAWK_GENERATOR_BLOCKS.md` ¬ß2.1,
   `UNIFYWEAVER_LANGUAGE_PRINCIPLES.md` Principle 2): **static type knowledge
   elides coercion.** It is layered *on* auto-coerce (the annotation-free
   default), not instead of it; an unannotated param still auto-coerces. In text
   mode a typed param is a contract that the arg is numeric ‚Äî a bare text field
   passed to it is the caller's responsibility. *Still open (follow-on):*
   compile-time **type checking** of call sites (a mismatched call ‚Üí error) and
   distinguishing `int` vs `float` for representation; all three keywords
   currently mean "numeric, skip coercion".
2b. **`if` with a plain guarded body + regex-in-`if` + braceless bodies ‚Äî
   LANDED.** `{ if (c) { print $1 } }`, `if ($0 ~ /re/) { ‚Ä¶ }` / `!~`, and
   if/else with plain bodies all compile and run (the guarded-print gap was
   fixed by the while-runtime body-print enablers ‚Äî a plain body no longer has
   to update a scalar). Braceless bodies also landed: `if (c) print`,
   `while (c) x++`, `do stmt while (c)`, and braceless else-if chains ‚Äî a
   control-flow body is a braced block *or* a single statement.
3. **`printf` format coverage ‚Äî LANDED.** The rewriter now parses the standard
   conversion prefix `%[flags][width][.precision][length]<conv>` and rewrites to
   a C printf spec driven by each argument's inferred kind: integer args (i64)
   take `d`/`i`/`x`/`X`/`o`/`u` (with the `l` length modifier) and `c` (code
   point ‚Üí character); float args (f64) take `f`/`g`/`e`/`F`/`G`/`E`; string
   args take `%s` with flags/width/precision. Flags `-+ 0#`, width, and
   precision are honoured. A record-field `%s` (a non-null-terminated slice)
   takes flags/width and rides `.*` for its length, so a user precision on a
   field is a clean compile error (buffer safety) ‚Äî a follow-on. `%c` needs a
   numeric arg (a string arg's first-char form is a follow-on). Tests:
   `tests/test_plawk_printf.pl`. *Still open (follow-on):* `%c` on a string
   (first char), precision on a field slice, `*`-width (width from an arg), and
   `printf` in a `BEGIN`/`END` block (a separate driver gap).
4. **`exit [n]` ‚Äî LANDED.** A rule-level `exit` / `exit N` stops the record loop,
   runs END, and returns N (default 0). It reuses the rule-level stream-break
   path (`break_close_stream` ‚Üí END ‚Üí `ret`), adding an exit code stored in
   `@plawk_exit_code` and read at the final `ret`. `exit` is modelled as a
   terminal control (`terminal_exit`, like `terminal_break`) at rule-body top
   level; inside an `if`/`else` branch or a loop it flows via the `branch_exit`
   exit + `plawk_branch_to_done_ir` (branch to `break_close_stream`). Unlike
   `break`, an `exit` inside a loop is NOT consumed by the loop ‚Äî it always ends
   the whole program (propagates past any enclosing loop). Scalar state at the
   exit point merges into END through the break-close phi. Tests:
   `tests/test_plawk_exit.pl`. *Still open (follow-on):* `exit` inside a `BEGIN`
   or `END` block, and non-constant exit codes (`exit code_var`).
5. **Ternary `?:` ‚Äî LANDED (print / printf, numeric).** `COND ? A : B` in a
   print field or printf argument: the condition is a numeric comparison
   `L <op> R` and both branches are numeric (fields, `NR`/`NF`, integer
   literals, i64 arithmetic). Lowered to an LLVM `select` ‚Äî both branches are
   evaluated (no side effects in an i64 expression), so it composes straight-line
   anywhere an i64 value is used (`plawk_i64_expr_ir(ternary(...))`), and the
   `%s`/`%d`/arith paths are unaffected. Tests: `tests/test_plawk_ternary.pl`.
   *Still open (follow-on):* assignment-context ternary (`x = c ? a : b`, needs
   scalar-var operands + the assignment RHS grammar), string-valued branches,
   and boolean-combination conditions (`a && b ? ...`).
   (String concatenation landed in both contexts ‚Äî `print $1 $2` earlier, and
   **assignment concat `x = $1 $2` via string-valued scalars** now: a string
   scalar's slot holds an interned atom id (an i64, so it reuses the whole
   SSA/phi scalar machinery), the RHS is built into a buffer and interned at
   assignment, and a read/print resolves the id to text ‚Äî id 0 is the unset
   sentinel, printed as empty. Numeric and string scalars coexist in one
   program. Tests: `tests/test_plawk_strscalar.pl`. *Still open (follow-on):*
   a scalar-var concat operand (`x = x $1` accumulation), string scalars in an
   `if`/loop body, and string comparison/guards on a string scalar.)
6. **`delete arr[k]` ‚Äî LANDED (field key).** `delete arr[$k]` removes the entry
   keyed by field k, matching the counted inc `arr[$k]++`. The runtime primitive
   `@wam_assoc_i64_delete` does **backward-shift deletion** on the linear-probing
   table, so later keys that collided into the same cluster stay reachable and
   `get`/`inc`/`set` need no change (the no-gap invariant they rely on is
   preserved); a missing key is a no-op. Wired through the assoc for-in driver
   (build ‚Üí delete ‚Üí for-in report). Tests: `tests/test_plawk_delete.pl` and
   `tests/core/test_wam_llvm_assoc_i64_runtime.pl` (backward-shift unit test).
   *Still open (follow-on):* string-literal / variable keys (`delete arr["k"]`,
   `delete arr[k]`), and `delete` in the scalar/mixed sequence-walker path.
7. **`split` / `sub` / `gsub` / `match` / `sprintf`** ‚Äî the string-builtin
   family; larger, lower priority for the DSL's data-pipeline focus.

Deferred by design (the DSL's model diverges here): multi-file `FILENAME`/`FNR`
(`ARGV[N]`/`ARGC` themselves are landed for rule
bodies ‚Äî see the table), C-style `for(;;)` / `do-while` (the `while`
runtime + for-in cover the loop needs).
