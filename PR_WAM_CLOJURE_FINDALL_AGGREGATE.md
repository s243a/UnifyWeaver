# PR Title

feat(wam-clojure): add sequential findall aggregate support

# PR Description

## Summary

- Add Clojure WAM runtime support for `begin_aggregate` / `end_aggregate` collection frames.
- Emit Clojure instruction maps for aggregate WAM instructions generated by `findall/3`.
- Keep aggregate predicates runtime-mediated for now instead of direct-lowering them.
- Add generator and runtime smoke coverage for `findall/3` over multi-clause facts, `member/2`, and empty result sets.

## Details

This implements the sequential `findall/3` substrate for the Clojure WAM target. The runtime now pushes an aggregate choice frame at `begin_aggregate`, collects copied template values at `end_aggregate`, forces backtracking through the goal body, and finalizes the collected values into a list when the aggregate frame is reached.

The lowered emitter recognizes aggregate instructions so aggregate predicates parse cleanly, but they remain on the runtime-mediated path until a dedicated lowered aggregate strategy is designed.
